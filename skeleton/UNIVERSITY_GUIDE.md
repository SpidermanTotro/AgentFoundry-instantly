# University Guide - AgentFoundry Skeleton for Education

## Introduction

Welcome educators and students! This guide explains how to use the AgentFoundry Skeleton in university courses, student projects, and research.

## Educational Use Cases

### Computer Science Courses

#### Software Engineering (CS 300-400 level)
- **Topic**: Software development lifecycle, automation
- **Use**: Repository health checks, CI/CD workflows
- **Projects**: Build automated quality assurance systems
- **Learning**: DevOps, workflow automation, testing

#### Artificial Intelligence (CS 400-500 level)
- **Topic**: Agent-based systems, AI development
- **Use**: Agent scaffolding, training utilities
- **Projects**: Develop intelligent agents
- **Learning**: Agent architectures, training pipelines

#### Human-Computer Interaction (CS 300-400 level)
- **Topic**: User interface design, web applications
- **Use**: Streamlit-based dev lab
- **Projects**: Build interactive development tools
- **Learning**: UI/UX design, web frameworks

#### Open Source Development (CS 400 level)
- **Topic**: Collaborative software development
- **Use**: Contribution workflows, automation
- **Projects**: Contribute to open-source projects
- **Learning**: Git workflows, code review, documentation

## Course Integration

### Assignment Ideas

#### Assignment 1: Agent Development
**Level**: Intermediate  
**Duration**: 2 weeks  
**Objective**: Create a functional agent using scaffolding tools

**Requirements**:
1. Use dev copilot to scaffold a new agent
2. Implement specific functionality (e.g., data processing, API integration)
3. Add comprehensive tests
4. Document the agent's capabilities
5. Package for distribution

**Grading Rubric**:
- Functionality (40%)
- Code quality (20%)
- Testing (20%)
- Documentation (20%)

#### Assignment 2: Automation Pipeline
**Level**: Advanced  
**Duration**: 3 weeks  
**Objective**: Build automated development pipeline

**Requirements**:
1. Set up repository health monitoring
2. Create custom GitHub Actions workflow
3. Implement automated testing
4. Add release automation
5. Document the pipeline

**Grading Rubric**:
- Pipeline completeness (30%)
- Automation effectiveness (30%)
- Error handling (20%)
- Documentation (20%)

#### Assignment 3: Template Development
**Level**: Intermediate  
**Duration**: 2 weeks  
**Objective**: Create reusable code templates

**Requirements**:
1. Identify common code patterns in a domain
2. Create at least 3 templates
3. Implement template variables
4. Test template generation
5. Document usage

**Grading Rubric**:
- Template usefulness (30%)
- Code quality (25%)
- Documentation (25%)
- Testing (20%)

### Lab Sessions

#### Lab 1: Getting Started (2 hours)
**Objectives**:

**Activities**:

### Rubrics

#### Code Quality Rubric
- **Excellent (90-100%)**: Clean, well-documented, follows best practices
- **Good (80-89%)**: Mostly clean, adequate documentation
- **Satisfactory (70-79%)**: Functional, basic documentation
- **Needs Improvement (<70%)**: Issues with functionality or quality

#### Documentation Rubric
- **Excellent (90-100%)**: Comprehensive, clear, well-organized
- **Good (80-89%)**: Complete, mostly clear
- **Satisfactory (70-79%)**: Basic documentation present
- **Needs Improvement (<70%)**: Incomplete or unclear

## Best Practices for Educators

### Course Setup
1. **Start early**: Setup environment before term starts
2. **Test assignments**: Complete assignments yourself first
3. **Provide examples**: Share sample solutions
4. **Offer support**: Hold office hours for questions
5. **Iterate**: Improve based on student feedback

### Student Support
1. **Clear instructions**: Provide detailed assignment descriptions
2. **Resources**: Share links to documentation
3. **Examples**: Demonstrate concepts in class
4. **Feedback**: Give timely, constructive feedback
5. **Help channels**: Offer multiple ways to get help

### Grading Efficiency
1. **Automated checks**: Use scripts to verify functionality
2. **Rubrics**: Use clear, detailed rubrics
3. **Peer review**: Have students review each other
4. **Templates**: Create grading templates
5. **Tools**: Use grading assistants
